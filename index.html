Intro

This is the example and documentation website for the zoon framework. I've designed it to be fully modular and fit most use cases. zoon is constantly evolving, changing and improving as I learn more about designing software. I'm primarily creating this framework for use in my own work so I can have stylistic consistency across all of the sites that I work on, but I plan for it to be customisable and approachable enough for others to use in their own creations.

zoon aims to be super simple to setup and get started with. All you need to do to start using it is add a link to the main style sheet in the head of your HTML file. This will instantly give you access to all of the different classes and stylings that zoon provides.

zoon_config.css

If you would then like to customise your zoon setup you can add the zoon_config.css template file locally and make changes to customise how zoon looks. This file is designed to let you make big changes easily without having to edit large parts of your HTML. It makes use of CSS variables that will override the default variables that the theme comes with. It is highly recommended that you use this file alongside zoon to get the most out of it.

The Pillars

Responsiveness

A site built with zoon should work across all devices and all browsers. Everything should have fallbacks available in case fonts or certain stylings are unavailable.

Contextuality

Not every part of a style should need to be applied manually. zoon will apply certain stylings for you automatically based on the context of what you're writing.

Presentation

Not only do I want zoon to function well, I want it to look good too. Using zoon it should be easy to build a site that works as well as it looks.

Style Rules

Fonts

Any fonts you're using can be set through the config file and are then automatically applied based on context using the default html elements.

Colours

Your color scheme is set inside of the config file and is then applied to various elements. This setup allows for easy changing of colour schemes without having the edit any classes

Borders

Values for borders can be altered in the config file. These values are used for the borders across all of the styling classes

Flexbox

zoon uses flexbox for properly sorting your content on the page. It allows content to be shown in a more interesting fashion and have a great level of responsiveness as it automatically changes to work with smaller viewports.

[zoon] Methodology

zoon works by properly seperating each part of your app to make development as intuitive as possible.
